# 국내 대학 학생지원 챗봇 사례 조사

- 조사 기준일: 2026-07-20 (KST)
- 조사 대상: 국내 대학이 공식 홈페이지·포털·앱·카카오톡 등을 통해 운영했거나 공개한 학생지원 챗봇
- 조사 방법: 대학이 공개한 서비스 소개, 공지, 사용 안내서, 현재 공개된 챗봇 화면을 우선 확인했다.
- 해석 표기:
  - **[확인]** 대학의 공개 자료에서 직접 확인한 사실
  - **[추정]** 화면·기능·설명으로부터 합리적으로 추정한 내용. 실제 구현과 다를 수 있다.
  - **[확인 불가]** 공개 자료에 근거가 없어 판단할 수 없는 내용

> 주의: `AI`, `인공지능`, `학습`이라는 홍보 표현만으로 생성형 AI나 특정 검색 아키텍처를 사용한다고 판단하지 않았다. 모델, 임베딩, 벡터 DB, RAG, 의도 분류기 등 기술 구성이 명시되지 않은 경우에는 확인 불가로 처리했다. 서비스 상태는 조사 기준일 이후 달라질 수 있다.

| 강냉이 에스크 적용 요소 | 학기 초·수강신청·시험·장학 시즌에 따라 **동적 퀵메뉴**를 교체하고, 자동완성·인기 질문으로 빈번한 질의를 빠르게 해결한다. 서비스 점검 때는 단순 오류 대신 FAQ·공지 검색·부서 연락처로 우회시킨다. |

## 비교 분석

### 공개 자료로 확인되는 설계 패턴

1. **공개 문서형 생성형 답변**: AI눈송과 SM-EDU는 대학 공식 문서를 근거로 대화형 답변을 만들고 출처 링크를 제공한다고 명시한다.
2. **인증 후 개인화 조회**: 고려대의 목표 기능, 서울사이버대 챗봇, 중앙대 e-Advisor, SM-EDU는 성적·출결·학습·학적 등 사용자별 데이터를 다룬다.
3. **FAQ·시나리오와 자연어의 혼합**: 서울사이버대의 인기 질문·바로 연결, 중앙대의 자동완성·TOP 답변·시기별 퀵메뉴, 가천대의 분야별 메뉴는 자유 입력만으로 해결하기 어려운 탐색 문제를 보완한다.
4. **사람·부서로의 전환**: 서울사이버대는 전화·추가 질문·카카오상담을, 가천대는 정보별 교내 연락처를 제공한다. AI가 답하지 못하는 경우의 종료 경로가 명확하다.
5. **근거와 한계 고지**: 숙명여대와 상명대는 출처 링크를 제공하고, 상명대와 사이버한국외대는 부정확성 및 재확인 필요성을 명시한다.
6. **채널 다변화**: 홈페이지·포털·앱·카카오톡을 사용한다. 반면 가천대 종료 사례는 특정 외부 채널 의존의 지속 가능성 위험도 보여준다.

### 공개 자료만으로 공통 확인이 어려운 부분

- 임베딩 모델, 벡터 DB, 키워드 검색, 재순위화, 의도 분류기 등 실제 검색 아키텍처
- 문서 수집·색인 갱신 주기와 오래된 공지의 만료·보관 정책
- 답변별 신뢰도·근거 선택 기준 및 환각 평가 지표
- 대화 로그, 인증 식별자, 개인 학사정보의 결합·보관·삭제 정책
- 답변 실패 시 담당 부서 티켓으로 전달되는 실제 운영 절차

따라서 위 요소는 특정 대학의 확정된 구현으로 인용해서는 안 된다.

## 강냉이 에스크 적용 제안

현재 강냉이 에스크는 공지 기반 답변, 하이브리드 검색, 담당 부서, 원문 링크, FAQ를 이미 MVP로 갖고 있다. 조사 결과를 반영한 우선순위는 다음과 같다.

1. **근거 표시를 제품의 중심으로 유지**
   - 답변마다 사용한 공지의 제목, 게시일, 상태, 담당 부서, 원문 링크를 표시한다.
   - 여러 공지를 합성했으면 근거를 모두 표시하고 문장 또는 단락별 인용 관계를 남긴다.
   - `공개 자료로 확인됨`, `추정`, `확인 필요`를 답변 표현에도 적용한다.

2. **시기별 퀵메뉴와 인기 질문 추가**
   - 중앙대처럼 수강신청·시험·장학·등록 기간에 맞춰 첫 화면 질문을 자동 변경한다.
   - 서울사이버대처럼 익명 집계한 인기 질문을 보여주되 개인 질문·식별자는 통계에서 제거한다.

3. **FAQ·학칙·공지의 통합 검색**
   - 상명대 사례처럼 부서 FAQ와 시행세칙을 공지와 함께 검색한다.
   - 자료 유형과 우선순위를 구분한다: `학칙/규정 > 최신 공식 공지 > 부서 FAQ > 안내 페이지`.
   - 서로 충돌하면 임의로 합치지 말고 충돌 사실과 최신 원문을 함께 제시한다.

4. **답변 실패 시 사람에게 안전하게 연결**
   - 검색 근거가 약하면 답을 생성하지 않고 담당 부서 카드, 전화번호, 운영시간, 문의 링크를 제공한다.
   - 사용자가 `상담 연결`을 선택한 경우에만 대화 요약을 전달하고, 전송 전에 내용과 개인정보를 검토하게 한다.

5. **공개 정보와 개인화를 분리**
   - 1단계는 로그인 없는 공개 공지·FAQ 검색으로 유지한다.
   - 2단계 개인화는 별도 인증 후 최소 권한 API로 `내 신청 상태`처럼 정형 조회만 수행한다.
   - 개인 데이터는 생성 모델의 장기 문맥·학습 데이터로 사용하지 않고, 목적·보관 기간·삭제 방법을 명시한다.

6. **검색 범위와 최신성 공개**
   - AI눈송 사례처럼 포함·제외 사이트를 사용자에게 보여준다.
   - `마지막 수집 시각`, `검색 대상 기간`, `첨부파일 분석 여부`를 답변 또는 서비스 정보에서 확인할 수 있게 한다.
   - 현재 MVP의 최근 2개월 수집 범위 밖 질문에는 범위 제한을 명시하고 전체 홈페이지 검색 또는 담당 부서로 안내한다.

7. **안전 문구와 질문 가이드 제공**
   - 개인정보·민감정보 입력 금지를 입력창 주변에 상시 표시한다.
   - 숙명여대처럼 `등록금`보다 `이번 학기 등록금을 분할 납부하려면 어떻게 해야 해?` 같은 구체적 질문 예시를 제공한다.
   - 마감·자격·금액처럼 영향이 큰 정보는 원문 재확인 경고를 자동 부착한다.

8. **운영 독립성과 장애 대응 확보**
   - 가천대 사례를 참고해 외부 메신저는 보조 채널로만 사용하고, 지식·로그·원문 링크의 기준 시스템은 대학이 통제한다.
   - 챗봇 점검 중에도 FAQ, 공지 검색, 담당 부서 디렉터리는 계속 이용할 수 있는 폴백 화면을 제공한다.

## 결론

공개 자료에서 가장 재사용 가치가 높은 조합은 **공식 문서 기반 생성형 답변 + 출처 링크 + 시기별 FAQ/퀵메뉴 + 담당 부서 전환**이다. 개인화는 편의성이 크지만 개인정보·권한 위험이 함께 커지므로 공개 검색과 분리해야 한다. 강냉이 에스크는 이미 공지 기반 하이브리드 검색과 원문·부서 안내를 갖추고 있으므로, 단기적으로는 검색 자료 유형 확대, 시기별 퀵메뉴, 실패 시 상담 전환, 검색 범위·최신성 표시를 우선 적용하는 것이 적합하다.
