# KNU-Ask 문제 해결 기록

이 문서는 주요 기술 변경을 기능 목록이 아니라 문제 해결 과정으로 정리한다. 세부 구현은 연결된 검증 문서와 Git 커밋에서 확인할 수 있다.

## 2026-07-22 · 구조화되지 않은 절차와 서류 복구

- **문제:** 관련 공지는 찾았지만 신청 순서, 제출 서류, 담당 연락처가 기존 구조화 필드에 없으면 안전한 답변을 만들 수 없었다.
- **원인:** 최초 수집 시 모든 예외 필드를 예측할 수 없고, 일반 HTML만으로는 PDF·이미지 첨부의 근거를 놓칠 수 있었다.
- **해결:** 실제 질문이 요구한 필드만 원문과 첨부에서 지연 재확인하고, 공식 도메인·원문 인용문·OCR 신뢰도를 통과한 사실만 `TaskFact`와 절차에 저장했다. 원문 해시가 바뀌면 이전 검증 결과를 무효화하고, 확인되지 않은 결과는 짧게 캐시했다.
- **검증:** 절차 3단계, PDF 서류, 업무 일치 연락처, 공식 신청 링크, 낮은 OCR 신뢰도 거부와 해시 무효화를 자동 테스트했다.
- **효과:** 첫 요청은 필요한 근거만 복구하고 이후 요청은 저장된 결과를 재사용해 PDF·OCR·Codex 호출을 반복하지 않는다.
- **근거:** [누락 정보 지연 재확인 검증](testing/lazy-evidence-recovery-benchmark.md)

## 2026-07-21 · 전체 선처리 비용을 온디맨드 검색으로 전환

- **문제:** 수천 건의 새·변경 공지를 모두 AI로 선처리하면 입력 토큰과 대기 작업이 크게 늘어났다.
- **원인:** 실제로 질문되지 않을 문서까지 같은 우선순위로 구조화하는 일괄 처리 방식이었다.
- **해결:** exact cache → QueryPlan → canonical cache → 기존 DB → 허용된 공식 사이트 → 부족할 때만 Codex 판정 순서의 온디맨드 파이프라인을 추가했다. 기존 일괄 워커는 기능 플래그로 되돌릴 수 있게 유지했다.
- **검증:** 캐시 적중, 원문 해시 변경 무효화, 공식 도메인 제한, 시간·URL·검색 횟수 상한을 회귀 테스트했다.
- **효과:** 자주 묻는 질문은 즉시 재사용하고, 비싼 추출과 모델 호출은 실제 근거가 부족한 질문으로 제한했다.

## 2026-07-21 · 크롤링 누락과 반복 수집 복구

- **문제:** 행사 0건, 하위 학사 탭 누락, 마지막 페이지 반복, NUL 문자 저장 실패, 확장자가 없는 이미지 인식 실패가 함께 발생했다.
- **원인:** 고정 시작점, 날짜만 이용한 종료 조건, 문자열 정제 범위 부족, URL 확장자에 의존한 파일 판별이었다.
- **해결:** 출처 프로필과 하위 탭 탐색, 페이지 지문 기반 반복 차단, 재귀적 NUL 정리, 파일 바이트 시그니처 판별과 첨부 해시 캐시를 적용했다.
- **검증:** 전체 수집에서 원문 최종 실패 0건을 확인하고, 17개 운영 질문 중 근거가 실제로 없는 1건만 의도적으로 `insufficient_evidence`로 남겼다.
- **효과:** 검색 범위가 공지·행사·학사·FAQ·요람·규정·학생지원·담당자 디렉터리까지 확대됐다.
- **근거:** [크롤링 확대 및 운영 검증](reviews/operational-validation-2026-07-21.md)

## 2026-07-21 · 의미 임베딩 호환성 회귀 방지

- **문제:** 테스트 벡터의 모델 식별자가 저장된 임베딩 모델과 달라 의미 검색 호환성 검증이 실패했다.
- **원인:** 테스트 fixture가 실제 검색 코드에서 요구하는 모델 버전 조건을 재현하지 못했다.
- **해결:** 테스트 AI와 문서·청크 임베딩에 동일한 모델 식별자를 지정했다.
- **검증:** 키워드가 겹치지 않는 의역 질문도 올바른 휴학 공지를 1위로 찾는 회귀 테스트가 통과했다.
- **효과:** 모델 버전이 다른 벡터를 섞지 않는 실제 동작을 테스트가 정확히 반영한다.
- **커밋:** `4aa9270 Fix semantic embedding compatibility test`

## 기록을 계속 추가하는 방법
